🛠️ 2. Các bước code chi tiết (Cầm tay chỉ việc)
  • Bước 1: Tạo mô hình dữ liệu (Models). Tạo các file Java thuần:
    - User.java (String username, String password, double balance).
    - Item.java (int id, String name, double startPrice).
    - Auction.java (Item item, double currentBid, String highestBidder, long endTime).
  • Bước 2: Viết class AuctionManager (Bộ não trung tâm). Áp dụng Singleton pattern để toàn hệ thống chỉ có 1 manager này. Dùng ConcurrentHashMap<Integer, Auction> để lưu danh sách các phòng đấu giá.
  • Bước 3: Viết hàm placeBid() (Quan trọng nhất).
    public synchronized boolean placeBid(User user, int auctionId, double amount) {
        Auction auc = auctions.get(auctionId);
        if (auc.isEnded() || amount <= auc.getCurrentBid()) return false;
        
        // Tính năng Anti-sniping (Gia hạn 30s)
        if (auc.getTimeLeft() < 30000) {
            auc.addExtraTime(30000);
        }
        
        auc.setCurrentBid(amount);
        auc.setHighestBidder(user.getUsername());
        return true;
    }
  • Bước 4 (Tính năng điểm A+): Viết hàm Auto-Bidding. Nếu User A cài giá tối đa là 10 triệu, hệ thống tự động gọi hàm đặt giá nâng lên 50k mỗi khi có người khác trả giá cao hơn, cho đến khi chạm mốc 10 triệu.

⚠️ Cảnh báo chết người:

Tuyệt đối không dùng ArrayList hoặc HashMap thông thường trong môi trường mạng nhiều người dùng. Hãy thay bằng CopyOnWriteArrayList và ConcurrentHashMap nếu không muốn bị lỗi văng app ConcurrentModificationException.

🌐 Thành viên 2: Kỹ sư Mạng & Máy chủ

Nhiệm vụ cốt lõi: Viết chương trình Server chạy ngầm, mở cổng kết nối, nhận tin nhắn từ hàng chục Client gửi lên và làm "tổng đài" phát loa thông báo cho mọi người.

📚 1. Giáo trình tự học (Search YouTube/Google các từ khóa này)

🔹 Kiến thức kết nối mạng

  • Socket Programming: Cách tạo ServerSocket đứng đợi và Socket đi kết nối.
    java socket programming client server example
  • I/O Stream: Đọc ghi chuỗi String qua mạng.
    java BufferedReader PrintWriter socket

🔹 Kiến thức quản lý Client

  • Thread Pool: Cách cấp cho mỗi Client 1 luồng riêng biệt để không ai phải đợi ai.
    java multithreaded server socket Example
  • Observer Pattern: Kỹ thuật "Phát thanh" (Broadcast).
    java chat server broadcast message to all clients
🛠️ 2. Các bước code chi tiết (Cầm tay chỉ việc)
  • Bước 1: Khởi tạo Server. Tạo file ServerMain.java. Dùng ServerSocket server = new ServerSocket(8080); và một vòng lặp while(true) để liên tục server.accept() (đón khách mới).
  • Bước 2: Tạo class ClientHandler (Người phục vụ riêng). Mỗi khi có 1 Client kết nối (khách vào), tạo ngay 1 đối tượng ClientHandler chạy trên 1 Thread mới để phục vụ riêng khách đó.
    *Lưu ý: Bạn phải tạo một biến List<ClientHandler> clients = new ArrayList<>() trên Server để quản lý danh sách toàn bộ khách đang online.
  • Bước 3: Định nghĩa Giao thức (Protocol). Quy định nhận gửi chuỗi theo chuẩn.
    - Nhận: "LOGIN|admin|123" -> Cắt chuỗi theo dấu | để xử lý.
    - Gửi: Dùng thư viện Gson (Google) biến Object Java thành chuỗi JSON gửi đi cho chuyên nghiệp (ví dụ: {"type": "UPDATE_PRICE", "price": 50000}).
  • Bước 4: Viết hàm Broadcast (Tính năng Realtime bắt buộc).
    public void broadcastMessage(String message) {
        for (ClientHandler client : clients) {
            try {
                client.sendMessage(message);
            } catch (Exception e) {
                // Nếu gửi lỗi (client rớt mạng), xóa client đó khỏi danh sách
                clients.remove(client);
            }
        }
    }

⚠️ Cảnh báo chết người:

Đừng bao giờ dùng lệnh đọc (ví dụ in.readLine()) ở luồng chính (Main Thread) của Server, nó sẽ làm Server bị "đứng hình" chờ đợi 1 client gõ phím mà bỏ quên hàng trăm client khác. Phải vứt nó vào Thread riêng!

📚 1. Giáo trình tự học (Search YouTube/Google các từ khóa này)

🔹 Kiến thức UI Cơ bản

  • JavaFX Scene Builder: Cài đặt phần mềm, cách kéo thả Layout (VBox, HBox, BorderPane).
    javafx scene builder tutorial for beginners
  • JavaFX MVC: Cách tạo file .fxml và file Controller.java.
    javafx fxml controller @FXML annotations

🔹 Kiến thức UI Nâng cao

  • TableView: Cách hiển thị danh sách sản phẩm lên bảng.

🛠️ 2. Các bước code chi tiết (Cầm tay chỉ việc)
  • Bước 1: Thiết kế giao diện "Tĩnh" bằng Scene Builder.
    Kéo thả 3 file: Login.fxml (ô nhập user/pass), Home.fxml (Bảng danh sách đồ đang đấu giá), AuctionRoom.fxml (Có hình ảnh món đồ, label giá tiền to đùng, ô nhập giá, nút BID).
  • Bước 2: Viết Controller bắt sự kiện.
    @FXML private TextField txtBidAmount;
    @FXML private Label lblCurrentPrice;

    @FXML
    public void handleBidButton() {
        String amount = txtBidAmount.getText();
        // Gọi class NetworkClient (của TV2) để gửi lệnh "BID|amount" lên server
        NetworkClient.send("BID|" + amount);
    }
  • Bước 3: Hiển thị giá Realtime (Cực kỳ quan trọng). Khi Class NetworkClient nhận được tin nhắn trả về từ Server là "Có người vừa đặt giá mới là 500", bạn KHÔNG được đổi text trực tiếp, mà phải bọc nó lại:
    Platform.runLater(() -> {
        lblCurrentPrice.setText("500.000 VNĐ");
        // Thêm hiệu ứng chớp màu xanh ở đây cho đẹp
    });

⚠️ Cảnh báo chết người:

Nếu bạn quên dùng Platform.runLater() khi cập nhật giao diện, JavaFX sẽ ném thẳng lỗi NotOnFXApplicationThreadException và văng app ngay lập tức. Hãy xăm câu lệnh này lên tay!

🔗 Ma Trận Ráp Nối (Toàn Đội Chú Ý)

Khi từng người đã code xong "đồ chơi" của mình ở tuần 1 và 2. Tuần 3 là lúc ngồi lại ghép thành cỗ máy hoàn chỉnh. Hãy ghép theo đúng thứ tự sau, cấm làm ngược lại!

BƯỚC 1: HỌP BÀN GIAO THỨC (TV2 & TV3)

Tại sao: Client (TV3) và Server (TV2) là 2 chương trình hoàn toàn riêng biệt. Cần thống nhất "ngôn ngữ" để hiểu nhau.

Hành động: Lập 1 file Excel chung. Liệt kê:
- Khi bấm nút Bid -> Client gửi: {"action": "BID", "itemId": 1, "amount": 500}
- Server xử lý xong trả về: {"status": "SUCCESS", "newPrice": 500}

BƯỚC 2: GHÉP DATABASE VÀO LÕI (TV4 & TV1)

Tại sao: Lõi Logic (TV1) cần dữ liệu thật từ ổ cứng để hoạt động.

Hành động: Trong file `AuctionManager` (của TV1), khi khởi động hệ thống (Constructor), hãy gọi hàm ItemDAO.getAllItems() (của TV4) để đổ toàn bộ hàng hóa từ Database lên RAM chuẩn bị đấu giá.

BƯỚC 3: GHÉP LÕI VÀO MẠNG (TV1 & TV2)

Tại sao: Server (TV2) nhận được lệnh "BID", nhưng Server ngu ngơ không biết tính toán, phải đưa cho Lõi Logic (TV1) tính.

Hành động: Trong class `ClientHandler` của TV2, khi nhận được chuỗi đặt giá, hãy gọi hàm auctionManager.placeBid(...) của TV1. Nhận kết quả True/False từ TV1 rồi Server mới Broadcast cho tất cả.

BƯỚC 4: GHÉP BIỂU ĐỒ VÀO UI (TV4 & TV3)

Tại sao: Đưa tính năng lấy điểm cao vào giao diện cuối cùng.

Hành động: TV3 chừa ra 1 khung (Pane) trống trong file FXML Phòng Đấu Giá. TV4 chèn cái LineChart vào đó. Gắn dây sự kiện: Mạng (TV2) báo có giá mới -> Giao diện đổi số (TV3) + Biểu đồ vẽ thêm đường lên (TV4).
